# Risk & Fraud Streaming Pattern

## Problem
Fraud and risk checks are often batch‑based and slow.

## Solution
Implement real‑time scoring pipelines using event streams.

## Components
- Event ingestion (Kafka)  
- Feature enrichment  
- ML scoring service  
- Decision event publishing  

## Benefits
- Sub‑second fraud checks  
- Reduced false positives  
- Continuous model improvement
